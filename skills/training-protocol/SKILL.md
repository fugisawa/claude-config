# Training Protocol — engine + domain modules

## Overview

One engine, swappable domains. A good personal protocol is **progressive load + real recovery + an unskippable daily anchor + honest timelines**, instantiated per domain (mobility, cardio, strength) *without* losing each domain's specificity. The most common failure is doing more, not better — adaptation needs supercompensation, not volume.

## When to use

- Building or revising a training protocol (mobility, cardio, strength) for one person.
- A daily tracker app to follow it (single-file, local, opens in the browser).
- Exercise cueing/clarity, periodization, or "is this too much / too little?"

**Not for:** diagnosis or rehab of an acute injury → a professional. Generic one-off gym tips.

## The engine — principles every module inherits

1. **Progressive overload, the right variable.** Mobility: controlled depth/range, then load. Cardio: volume, then intensity. Strength: load/reps. Advance one variable at a time.
2. **Recovery is the training.** Adaptation happens *between* hard sessions. Fewer quality sessions + easy/anchor days beat daily maximal effort. Never two hard days on the same system back-to-back.
3. **The daily anchor.** A ~5-min non-negotiable (mobility: CARs; cardio: easy movement; strength: mobility prep) that keeps the habit alive on bad days. Rule: never two days without touching the system.
4. **Specificity.** Train the actual target pattern/energy system. Range you can't control isn't yours; an aerobic base you didn't build won't appear at threshold.
5. **Autoregulation.** Scale to readiness (sleep, soreness, life). RPE/pain caps over fixed numbers. **Pain is a stop signal, never a target.**
6. **Measure & film.** Re-test every ~4 weeks from the side / by the numbers (ROM photos, HR at a fixed pace, loads). Proprioception lies; the camera and the watch don't.
7. **Warm, then load.** No max range or intensity cold. Highest-risk work only when tissue is prepared.

## Instantiate a protocol

1. Assess the **honest starting point** (what's missing, not what's strong).
2. Pick 4–5 focused sessions + the daily anchor; separate hard sessions of the same system.
3. Phase it (e.g. Pattern → Load → Expression → Maintain) with **honest** timelines and an anatomy/biology caveat where ceilings are individual.
4. Cue each item: **dose, equipment, the ONE cue that matters, the error to avoid**, and a **YouTube-searchable name** (official, or in parentheses) so it can be looked up.
5. Build the tracker → `tracker-app.md`.

## Domain modules

- **Mobility / flexibility** → `module-mobility.md` (PAILs/RAILs, loaded stretching, CARs, deficit→arsenal).
- **Cardio / conditioning** → `module-cardio.md` (HR zones, Z2 base, 80/20 polarized, VO2max intervals).
- **Strength** → extend the same pattern (compound lifts, progressive load, ≥48h between same-pattern sessions, depth/control before weight).

Modules compose: e.g. an athlete on a hot-yoga base adds loaded mobility (5×/wk) + Z2 cardio (easy, low recovery cost) with one interval slot that gets its own recovery day.

## Tracker app

A single-file `index.html` that opens in the browser and saves progress between sessions. Pattern + the hard-won UX/legibility lessons: `tracker-app.md`.

## Safety

- Joint pain = stop, not push. Distinguish stretch discomfort from joint pain.
- Get **medical clearance** before new high-intensity cardio or loaded end-range work if there's any cardiovascular or joint history.
- Individual anatomy caps some end ranges (hip morphology, etc.). The goal is **safe capacity, not a forced shape**.
